Original Artifact

The original version was created in CS-210 as a C++ console inventory tracker. It demonstrated core programming concepts and basic data handling in a command-line environment.

  • Console-based input/output
  • Basic inventory tracking logic
  • Limited modularity and no persistence

Enhanced Artifact (Web App)

The enhanced version is a modular web application designed to run on GitHub Pages. It includes a structured UI, separation of concerns (UI/service/database), custom sorting logic, persistent storage via IndexedDB, and a permission gate.

  • Search, sort, and CRUD operations
  • Merge sort and in-memory indexing
  • IndexedDB persistence with unique SKU index
  • Read-only default mode with admin unlock

Enhancement 1: Software Design and Engineering

The first enhancement focused on transforming the artifact into a maintainable, modular application with a clear UI. I separated responsibilities into logical modules and created a layout that better fits a real-world user workflow. This enhancement improved readability, reusability, and long-term maintainability.

  • Modular structure (UI layer + service layer)
  • Improved usability and clearer workflow
  • More professional formatting and comments

Enhancement 2: Algorithms and Data Structures

The second enhancement focused on algorithmic reasoning and efficiency. I implemented a custom merge sort algorithm to handle sorting consistently with predictable O(n log n) performance. I also introduced Map-based indexing for fast SKU lookups and to support uniqueness checking efficiently.

  • Merge sort for sorting by name, SKU, quantity, or price
  • Comparator-based sorting design
  • Map-based indexing for faster lookup and validation support

Enhancement 3: Databases

The third enhancement introduced persistence and database concepts using IndexedDB, which is a native browser database suitable for static hosting on GitHub Pages. I defined an object store schema, added a unique SKU index to enforce integrity, and implemented transaction-based CRUD operations. The service layer maintains consistent state between the UI and the database.

  • IndexedDB database setup (object store + primary key)
  • Unique SKU index to prevent duplicates
  • Transaction-based create, read, update, and delete
  • Permission gate to restrict writes in read-only mode
